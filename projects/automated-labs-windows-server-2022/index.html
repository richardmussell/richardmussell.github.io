Automated Labs: Provisioning a Windows Server 2022 VM with PowerShell ISE

Jul 21, 2025

In this project, I leveraged the power of AutomatedLab—a powerful PowerShell module designed to automate the deployment of virtual lab environments—to provision a fully functional Windows Server 2022 virtual machine. Using PowerShell ISE as my scripting environment, I was able to streamline the setup process, saving valuable time while ensuring consistent, repeatable results.

Why AutomatedLab?

Setting up virtual labs manually can be tedious and error-prone, especially when working with complex environments involving multiple servers, domain controllers, and network configurations. AutomatedLab abstracts much of the heavy lifting, allowing IT professionals and learners to deploy entire lab topologies with just a few lines of PowerShell code.

This project was focused on:

  • Creating a Windows Server 2022 VM configured as a Domain Controller
  • Structuring Organizational Units (OUs) and Active Directory (AD) objects for realistic testing
  • Automating Group Policy Object (GPO) linking and management
  • Ensuring the environment was suitable for hands-on learning and experimentation

Tools Used

  • AutomatedLab PowerShell Module: To define and provision virtual machines, networks, and roles
  • PowerShell ISE: Script editing and debugging environment for authoring the deployment scripts
  • Hyper-V: Virtualization platform hosting the lab VMs
  • Windows Server 2022 Datacenter: Target OS for the domain controller VM

Step-by-Step Process

1. Defining the Lab Environment

Using AutomatedLab's New-LabDefinition cmdlet, I created a lab definition specifying the lab name, virtualization engine (Hyper-V), and the base OS image:

New-LabDefinition -Name 'ADLab' -DefaultVirtualizationEngine HyperV

Add-LabIsoImageDefinition -Name 'WinServer2022' -Path 'D:\ISO\WindowsServer2022.iso'

2. Adding the Domain Controller Machine

Next, I defined the domain controller VM, allocating 4GB of RAM and specifying the OS:

Add-LabMachineDefinition -Name 'Lab1' -Memory 4GB -OperatingSystem 'Windows Server 2022 Datacenter Evaluation (Desktop Experience)' -Roles RootDC

3. Customizing the Active Directory Structure

After provisioning, I scripted the creation of realistic Organizational Units (OUs), users, and groups to mirror a typical enterprise environment. This step involved PowerShell cmdlets like New-ADOrganizationalUnit and New-ADUser to simulate various departments and roles.

4. Automating Group Policy Objects (GPOs)

With the lab environment in place, I linked relevant GPOs to OUs using PowerShell, enforcing security baselines, password policies, and access controls necessary for real-world scenarios.

5. Testing and Validation

The lab was tested thoroughly to ensure AD replication, user authentication, and policy enforcement functioned as intended. The environment supports experimentation with security tools, incident response exercises, and system administration training.

What I Learned

This project deepened my understanding of:

  • Infrastructure-as-Code principles by automating lab deployment
  • PowerShell scripting best practices for managing AD environments
  • Practical setup of a Windows Server 2022 domain controller in a virtualized lab
  • The value of repeatable, scalable lab environments for IT training

Why This Matters

Automated, realistic labs empower IT professionals to experiment, learn, and validate configurations without risking production systems. Mastering tools like AutomatedLab and PowerShell equips admins and security analysts with skills directly applicable to enterprise environments.
